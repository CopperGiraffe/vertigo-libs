Gestion des transactions Dynamo.

Une transaction Dynamo est constituée de ressources.
Ces ressources représentent des connexions à une base de données, des mailers etc...
Les transactions sont fournies par KTransactionManager.

Une transaction est :
  • soit en cours
  • soit terminée
Il est possible de commiter ou rollbacker les transactions en cours. Cette action se propage sur toutes les ressources participant à la transaction.
Il est aussi possible de rollbacker une transaction déjà terminée, en ce cas il ne se passe rien.

Les transactions Dynamo sont conçues comme un liste ordonnée finie de ressources.
Les ressources doivent être placées dans la transaction de la plus critique à la moins critique.
