Gestion des services Vertigo.

Introduction

Un service, au sens Vertigo, fait partie de la couche Intégration ou DAO (Data Access Object) et permet de faciliter les échanges avec n'importe quel système de stockage de données :
  • Base de données relationnelle,
  • LDAP,
  • Base de données non relationnelle,
  • Moteur d'indexation,
  • Web Services,
  • etc...

Vertigo est fondé sur une approche déclarative, les services Vertigo sont donc conçus pour centraliser la problématique d'accès aux données en un et seul endroit (Fichiers de paramétrage KSP). La déclaration des services est une tache de conception.

Principes

Pour utiliser un service Vertigo, il faut l'avoir préalablement paramétré. Un service Vertigo doit être vu comme une boîte noire réalisant une tache bien précise. Les services Vertigo sont des services techniques et non métier, ainsi ne faut-il pas les comparer avec les services métiers généralement évoqués dans des architectures SOA (Services Oriented Architecture). Comprendre un service c'est comprendre son utilisation et son paramétrage.

Utilisation

Le fonctionnement d'un service se passe en trois étapes
  • injection des données,
  • Exécution du service
  • Récupération des données créées.

Le fonctionnement est monitoré de façon automatisé.
  • Fonctionnement réussi ou non,
  • Temps de réponse

Paramétrage

Le paramétrage est un travail de conception qui permet de préciser :
  • les données en entrée (nom, domain vertigo, caractère obligatoire),
  • les données en sortie (nom, domain vertigo, caractère obligatoire),
  • le moteur d'exécution utilisé : ServiceProvider,
  • le paramétrage du ServiceProvider.

Concrètement si l'on désire effectuer une requête à la base de données (type de recherche de personne à partir d'un formulaire de recherche), on précise :
  • Le nom du Service ex: SV_LOAD_XXXX
  • Le serviceProvider à utiliser ex: ServiceProviderSelect
  • Les paramètres en entrée ex: le critère de recherche nommé DTO_CRITERE de type DT_PERSONNE_CRITERE
  • Les paramètres en sortie ex: la collection DTC_RESULT de type DT_PERSONNE
